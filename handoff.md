## Conventions / tooling

- Commands: `npm run typecheck`, `npm run lint`, `npm run lint:fix`,
  `npm run format`, `npm run format:fix`, `npm run test` (Vitest),
  `npm run test:e2e` (Playwright), `npm run dev`, `npm run deploy`.
- Commit messages: lowercase, short, trailing period (match existing history).
- `worker-configuration.d.ts` is gitignored and regenerated by `npm run typegen`
  (run after changing `wrangler.jsonc`). Do not commit it.
- Server-only crypto tests must use `// @vitest-environment node` (jsdom breaks
  WebCrypto `instanceof` checks) - only relevant if JWT verification returns.
- Do not commit/push, merge PRs, or change Cloudflare config without explicit
  user confirmation.

## Guardrail before importing real data

Access security depends on config discipline: keep `workers_dev: false`, don't
add another route/custom domain without its own Access app, and don't re-enable
preview/version URLs. If any un-gated path is introduced, restore the in-Worker
JWT verification (from `phase-1-access` history) so it fails closed.
